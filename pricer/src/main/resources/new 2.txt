3. Алгоритм
{START}
3.0. Считать App конфигурацию.
3.0.1. По списку путей считать конфигурацию прайсов, каждая из которых состоит: корректного XML; наличие прайса xlsx; новой хеш-суммы; 

3.0.2. В цикле пройтись по всем файлам:
{Процесс индивидуального прайса}
		3.1. Привести все файлы к единому формату.
		3.3.1. Для одного файла считать конфигурацию из XML из одноименной папки другого рута.
		3.3.2. Вычитать содержимое из ексель файла в объект PriceBook. (ReaderService)
		3.3.3. PriceBookRecord состоит:
			3.3.3.1. цена розница;
			3.3.3.2. цена отп;
			3.3.3.3. цена акция;
			3.3.3.4. количество;
			3.3.3.5. наличие (булевское);
			3.3.3.6. номер строчки в файле поставщика.
			3.3.3.7. поставщик+артикул у поставщика.
		3.3.4. Для нового PriceBookRecord (появившийся из p0 в p0_result_new) проверить наличие ID (сайта) из c0/p0.props и заполнить колонку ID, иначе - пусто.
		3.3.5. Если ID для новой записи найден - обновить поле "не определен" в false.
		3.3.6. Для каждой добавленой записи поставить поле новый = true.
		
3.3.4. Считать существующий p0_result, обновить данные из полученного PriceBook (количество, цена, append новых строк). Записать в файл p0_result_new.xlsx
{Процесс сводного файла}
3.4. Открыть пропертис файл с  Привязкой поставщик+артикул у поставщика к ID Андрея (1 : 1).
3.4.1. В цикле пройтись по всем файлам
		3.4.2. Добавить итеративно все новые (поле новый == true) строчки из каждого p0_result_new в result_new
		3.4.3. Получить из файла props ID по (поставщик+артикул), записать ID в строчку в файле result_new. Если не найден - поставить флаг "не определен" = true.
		3.4.4. Если ID определен - записать в файл c0/p0.props SLN (source line number) и ID.
		3.4.5. Если ID определен - записать в файл p0_result_new ID по номеру SLN. Поле новый в файле p0_result_new поставить = false.
		3.4.6. Поле новый в result_new поставить = true
3.5. В случае успеха записать файлы с суфиксом "new" в аналогичные файлы без суфиксов.
{FINISH}
